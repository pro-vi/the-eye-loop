# Eye Loop Fracting: Hierarchical BALD and Orthogonal Decomposition for Intent Discovery Under a Swipe Budget

## Fracting as conditional search in a variable-dimensional space

“Fracting” (your term) is a precise problem class: once the user has **committed** to a region (“dark atmospheric”), the system must discover **the independent degrees of freedom that remain inside that region** (texture, density, motion, temperature…) and then allocate queries to resolve *those*—without “un-resolving” the parent concept.

That is structurally identical to **conditional / hierarchical configuration spaces** in Bayesian optimization and AutoML: some parameters only “exist” (are relevant) after a parent choice is made (e.g., choose optimizer = Adam → Adam-specific parameters become active; choose a model family → family-specific hyperparameters become active). These are explicitly treated as **hierarchical / conditional search spaces** in practical HPO systems and research, because the effective dimensionality **changes with the branch.** citeturn11view1turn7search1turn7search12turn7search17

Two points from the conditional-parameter-space literature map cleanly to Eye Loop’s fracting:

- **“Relevance masks” are first-class.** Kernels / similarity functions should encode which parameters are relevant under a branch; otherwise observations from one branch contaminate another. A well-known approach is to define kernels that incorporate parameter relevance so that configurations on different “inactive parameter” branches are treated as dissimilar. citeturn11view1turn9search11  
- **Tree-structured spaces are not a metaphor—people literally model them as trees.** TPE (tree-structured Parzen estimator) explicitly models the search space as a **tree-structured generative process**, which is exactly “top-level axis → sub-axes only if chosen.” citeturn7search4

So: fracting is not “extra UX”; it’s **the core move that makes a log-query story plausible in a high-dimensional aesthetic space**. If you don’t explicitly treat the space as conditional/hierarchical, you either (1) waste queries exploring irrelevant sub-axes or (2) smear evidence across incompatible regions and your posterior stops making sense. citeturn11view1turn7search4turn7search1

## Hierarchical BALD via conditional mutual information

### The right generalization: conditional mutual information under a resolved parent

BALD is an expected information gain criterion, i.e., a mutual-information objective used as an acquisition function. Modern “unification” treatments describe BALD as a specific instance of expected information gain (EIG) in Bayesian experimental design / active learning. citeturn0search9

To extend BALD from a flat space into a fract tree, the minimal formal move is to replace “mutual information” with **conditional mutual information**:

For a node \(v\) in the fract tree, let \(c_v\) denote the already-resolved parent commitments (the path conditions). Let \(\Theta_v\) be the node’s local preference parameters (boundary shape, local utility weights, etc.). Let \(y\) be the observation (binary choice and possibly latency). The node-conditional BALD acquisition is:

\[
\text{BALD}_v(x)
= I(\Theta_v;\, y \mid x, D, c_v)
= H(y \mid x, D, c_v)\;-\;\mathbb{E}_{\Theta_v \sim p(\Theta_v\mid D, c_v)}\big[H(y \mid x, \Theta_v, c_v)\big]
\]

This is exactly the quantity computed in multiple “infomax / mutual-information” active learning lines, where the objective is explicitly conditional MI between responses and parameters given the stimulus. citeturn14view1turn14view0turn0search9

Two supporting precedents matter for Eye Loop:

- **Conditional MI as the explicit objective in stimulus selection.** In active learning for discrete latent-variable models, stimuli are chosen to maximize *conditional mutual information* between response and parameters. citeturn14view1  
- **Conditional MI as a query criterion in classic active learning.** Earlier work explicitly frames query selection as maximizing conditional MI over labels given the current state of knowledge. citeturn14view0  

So the “hierarchical BALD” answer is not exotic: it’s **BALD-with-context**, where the context is the resolved parent path \(c_v\).

### Variable dimensionality: treat “fract nodes” like conditional parameter branches

In conditional hyperparameter spaces, two branches can have different active dimensions; to compare them safely, you either (a) embed them into a common space with relevance masks, or (b) use branch-aware kernels / partitioning models. citeturn11view1turn7search1turn7search4turn7search12

Eye Loop’s analogue:

- Each resolved parent axis activates a **different local coordinate system** for “what variations matter next.”  
- Therefore, your belief state should be **factored** by branch: \(p(\Theta \mid D)\) becomes something like \(p(\text{branch }v \mid D)\, p(\Theta_v \mid D, c_v)\).  
- Query value inside a branch should be **weighted** by the posterior probability that the branch is the user’s current “home region.”

This is the same structural rationale behind conditional kernels for BO: observations in one conditional branch shouldn’t leak into another unless you have an explicit mechanism for transfer. citeturn11view1turn7search1turn9search11

### Implementation consequence: “fract or surface?” becomes a conditional-VOI comparison

A flat BALD policy picks the single \(x\) with highest MI. A fract-tree policy must pick both:

1) which node \(v\) (surface-level axis exploration vs deeper fracting), and  
2) which stimulus \(x\) within that node.

The clean decision-theoretic template is:

\[
(v^\*, x^\*)=\arg\max_{v \in \mathcal{F}}\;\frac{p(v\mid D)\,\max_{x\in \mathcal{X}_v} I(\Theta_v;\, y \mid x, D, c_v)}{\text{Cost}(x)}
\]

where \(\mathcal{F}\) is your current frontier of “eligible-to-probe” nodes, and Cost can be modeled as expected cognitive time/effort (reaction-time predicted, or a constant per query). Conditional-MI computation and cost-normalized EIG are both standard moves in Bayesian experimental design implementations. citeturn14view1turn0search9turn6search7

(If you *don’t* do the \(p(v\mid D)\) weighting, you’ll occasionally dive deep into a branch the user hasn’t actually committed to—i.e., beautiful math, terrible product.)

## Orthogonal decomposition inside a resolved region

Fracting only works if you can **discover the hidden sub-axes** rather than invent plausible-sounding ones. You asked for “implementation-ready”: below are the most direct, formal toolkits that map to your setup.

### Local tangent-space estimation as “the sub-axes that actually exist here”

If your artifact space is a learned latent space (image or UI), then “dark atmospheric” corresponds to a **local neighborhood** of points. If that neighborhood lies on a lower-dimensional manifold, then the principled way to find “what directions exist here” is to estimate the **local tangent space**.

A standard method is **local PCA** on samples from a neighborhood; theoretical analyses study when PCA on a neighborhood estimates the tangent space of a smooth manifold and how sampling/noise affect accuracy. citeturn2search7turn2search0

Eye Loop translation:

- Take the accepted anchor \(z_0\) (fast accept).  
- Generate a small cloud of candidates \(z_0 + \delta_i\) (carefully, so they remain in-region).  
- Use the *accepted* subset (and optionally “near-boundary” slow rejects) to estimate local covariance, then take top eigenvectors as candidate “axes that the generator can vary smoothly here.”

This gives you **data-derived axes** rather than “LLM vibes.”

### Conditional / residual PCA: hold resolved dimensions constant, extract remaining variance

If you already know the parent axis is “dark atmospheric,” you want to remove (regress out / condition on) that factor and decompose what’s left. In multivariate analysis that’s exactly the point of “PCA with respect to instrumental variables / covariates” style approaches: separate sources of variation so residual structure becomes visible. citeturn15search4turn15search1

In Eye Loop terms, the practical move is:

- Fit a lightweight model predicting “dark atmospheric score” (even a linear probe in embedding space).  
- Compute residuals of candidates after removing that component.  
- Do PCA / factor analysis on residuals to identify orthogonal directions that remain. citeturn15search4turn15search1

If you want probabilistic uncertainty on those directions (useful for “do we know this axis exists?”), probabilistic PCA gives a likelihood-based frame instead of purely geometric eigenvectors. citeturn15search17

### Independence objectives: ICA / independent subspaces for “actually orthogonal” factors

PCA gives orthogonality in the *Euclidean* sense (uncorrelated under a covariance model), but you asked for *independent* sub-axes. That is where ICA-family or independent-subspace methods become relevant:

- Independent Subspace Analysis and related work explicitly target learning factors where each subspace is independent (often using non-Gaussianity / priors to break rotational ambiguity). citeturn2search19  
- Newer methods like supervised independent subspace PCA explicitly add independence constraints (e.g., using HSIC) while learning multiple subspaces. citeturn2search2turn2search5

Eye Loop translation: if you have enough local samples (accepted/near-boundary), you can try to learn **multiple independent directions** that each predict preference variations inside the region, instead of one tangled “style vector.”

### Conjoint-style interaction discovery: sub-axes that only matter in combination

You explicitly called out “dimensions that only matter in combination.” That is not a PCA problem; it’s an **interaction** problem.

Conjoint / factorial experiment methodology treats attribute interactions as causal interaction effects: combinations that produce effects beyond the sum of individual attribute effects. This is formalized in causal interaction analysis for factorial/conjoint settings. citeturn2search20turn2search12

Eye Loop implication: when fracting reveals candidates like {texture, motion}, you should sometimes probe **pairwise interactions** rather than single-axis sweeps. A minimal design is:

- pick two candidate axes \(a,b\)  
- generate a 2×2 (low/high) factorial micro-batch (or a small Latin hypercube in that local plane)  
- measure whether the user’s responses show a non-additive pattern.

Adaptive choice-based conjoint analysis is also a strong precedent for tailoring later questions based on early answers—explicitly motivated by engagement and utility estimation improvements. citeturn14view2turn12view0

### Connecting back to negative selection: rejects inside a “self region” reveal internal boundaries

You already established negative selection: self emerges from accumulated rejections (boundary learning). The missing fracting connection is:

- Once the parent region is “self,” **local rejections** carve *internal* forbidden subregions.  
- The geometry of those rejections (which directions produce fast reject vs slow reject) is exactly the signal you need to discover sub-axes: directions where tiny steps flip acceptance quickly are “active” axes; directions where you can move far without changing acceptance are “slack” axes.  
- This is naturally expressible as *local sensitivity / local boundary curvature* estimation, and tangent-space methods give you the basis to do it. citeturn2search7turn14view1

## Prompting multimodal models for orthogonal variation (and where it breaks)

You asked two things here:

- Can LLMs reliably decide orthogonality vs correlation?  
- How do you prompt Gemini to vary one axis while holding resolved dimensions constant?

### Why “orthogonal concepts” are hard for LLMs in the first place

In controllable generation research, two failure modes are repeatedly emphasized:

1) **Multi-attribute control is hard** because attributes co-occur in real corpora; models internalize those correlations, creating “stereotypes” where changing one attribute unintentionally drags others. That’s explicitly discussed in multi-aspect controllable text generation work: imbalanced attribute correlations lead to unwanted coupling, harming control of rare combinations. citeturn16view1  
2) Controllable Text Generation surveys classify control into content control vs attribute control, and note that real-world constraints often require complex combinations while preserving fluency/quality—i.e., “hold constant while varying one axis” is precisely a known hard case. citeturn16view0  

For Eye Loop, that means: **you should assume LLM-generated “axes” will be correlated by default** unless you build a mechanism that (a) tests axes empirically (with user feedback), or (b) imposes independence constraints post hoc (via your own decomposition and rejection learning).

### Disentanglement principles you can steal (even without training your own VAE)

You referenced β-VAE and InfoGAN; their core insight is: disentanglement comes from **explicit objectives**, often information-theoretic ones.

- InfoGAN adds a mutual-information term so that certain latent codes control interpretable factors. citeturn16view3turn3search5  
- β-VAE and follow-up analyses argue disentanglement emerges under constraints on latent channel capacity and independence, interpretable via rate–distortion perspectives. citeturn16view4turn3search16  

Eye Loop prompting analogue: you don’t have direct access to latents/attention the way diffusion-control papers do, but you can imitate the **structure**:

- define a *control variable* (your axis) with discrete levels  
- demand that outputs differ *only* along that axis  
- require a self-check description that explicitly lists invariants and the single changed factor  
- use user feedback to “score” whether the axis stayed isolated, and penalize drift (closed-loop control).

This is essentially: “use the Eye Loop itself as the disentangler,” which is consistent with the CTG framing that control can be applied at inference via prompt engineering and decoding-time constraints rather than retraining. citeturn16view0  

### Gemini-specific: use reference-image editing to enforce invariants

The best practical mechanism for “hold resolved dimensions constant” in Gemini is **image editing / inpainting-style prompting with a reference image**, not pure text-to-image from scratch:

- Gemini’s image generation docs describe that Gemini can generate and process images with text and image inputs, supporting iterative editing. citeturn11view4turn8search0  
- Official guidance for Gemini 2.5 Flash Image explicitly recommends prompts of the form “Keep everything else exactly the same, change only X,” and describes inpainting-style edits that touch only a specific element while preserving composition/lighting. citeturn17view1  
- Google Cloud’s “Nano Banana” prompting guide emphasizes maintaining consistency by combining **multiple reference images** plus a “relationship instruction,” precisely for character/style consistency. citeturn11view5turn8search17  
- Vertex AI documentation shows the “edit images with Gemini” workflow and model identifiers for image editing. citeturn17view3turn8search6  

In Eye Loop language: once the user “locks” the parent concept, you should pass the current best artifact as a **reference**, and ask Gemini to generate a *set of controlled edits* that vary only one axis. This is the closest you get to “semantic orthogonality” without internal model surgery.

## Depth vs breadth: the fract decision criterion as bandits + tree search + AMPLe-within-node

You asked for an explicit mapping. Here it is, mechanically.

### Tree search framing: nodes are regions; children are fracts

Model the Anima not as a flat boundary but as a **tree of conditional regions**. Each node \(v\) has:

- a posterior mass \(p(v\mid D)\): “how likely is it the user is in this region?”  
- a local belief \(p(\Theta_v\mid D, c_v)\) over the boundary/utility inside the region  
- an action generator that can propose stimuli \(x\in \mathcal{X}_v\)

Choosing “go deeper” means **expanding a child** (creating a new conditional sub-axis and probing it). Choosing “go broader” means **probing other high-level nodes** to reduce uncertainty about which region is correct.

This is not unlike MCTS: selection chooses a node; expansion adds children; simulation/evaluation estimates reward; backprop updates statistics. Surveys emphasize techniques like **progressive widening** for large action spaces: initially explore few actions, then expand more as the node becomes “worth it.” citeturn0search12turn0search5turn0search16

Eye Loop takeaway: fracting is literally “progressive widening” of the preference-action space—only add sub-axes when evidence says the parent region is promising.

### Bandit interpretation: reward = information gain (or IG per cost)

Multi-armed bandit logic maps cleanly if you treat each frontier node \(v\) as an “arm,” and the “reward” of pulling that arm as expected information gain (or expected reduction in posterior entropy) per swipe.

To balance exploration vs exploitation, classic UCB-style algorithms choose arms by an optimism-in-the-face-of-uncertainty rule; modern references treat this as foundational for the exploration/exploitation tradeoff. citeturn6search18turn6search5

For a *tree* of arms, hierarchical optimistic optimization (HOO) explicitly builds and refines a tree where refinement happens more near high-payoff regions, with regret bounds under local regularity assumptions. The core point: **the tree gets refined where it matters**, not uniformly. citeturn11view2turn6search8

This is extremely fract-compatible: “dark atmospheric” becomes a refined region because user signal makes it high-value; other regions stay coarse.

### AMPLe-within-node: generalized binary search as the local query policy

AMPLe provides a concrete, implementable active-query rule that tries to pick comparisons that split belief roughly in half (generalized binary search behavior). Their acquisition chooses the query whose two possible feedback outcomes are closest to 0.5 under the current belief—i.e., “make the next answer maximally informative.” citeturn13view1turn13view3turn11view0

Inside a node \(v\), you can apply the same logic:

- treat the node’s “sub-axis hypothesis set” as \(\Omega_v\)  
- pick stimuli whose accept/reject likelihood is near 0.5 under the node posterior (or whose BALD value is maximal)

This is your “BALD meets hierarchical exploration” convergence point: **global decision = which node; local decision = best query in that node.** citeturn0search9turn13view1

### Diminishing returns at depth: formal and practical signals

Even without deriving a new theorem, you get three concrete “stop going deeper” flags from the literature mechanics you already adopted:

- If within-node queries no longer offer near-0.5 splits (i.e., marginal likelihood becomes highly imbalanced), then AMPLe-style halving stalls and query efficiency drops. citeturn13view1  
- If conditional MI inside a node collapses (posterior already concentrated), then EIG is low and the bandit reward of deeper pulls goes down. citeturn14view1turn0search9  
- If the effective action set gets huge, MCTS-style methods recommend progressive widening—don’t expand deeper unless the parent node has been visited enough to justify it. citeturn0search12turn0search5turn0search16  

(Translation: if you try to fract six levels deep after five swipes, you’re not “amplifying taste,” you’re speedrunning confusion.)

## Hierarchical JND and practical dimensionality limits

### JND is not flat: commitment can sharpen some distinctions and blur others

The psychophysics story that matters for fracting is not just “there’s a floor,” but **what happens after categorization/commitment**.

Category-learning research shows a reliable pattern:

- Dimensions relevant to category discrimination can become **more distinctive** (“acquired distinctiveness”), improving discriminability along relevant axes, sometimes especially near boundaries (categorical perception effects).  
- Dimensions irrelevant to discrimination can become **less distinctive** (“acquired equivalence”), reducing discriminability along irrelevant axes. citeturn18view3turn18view4turn4search9  

That is a hierarchical-JND mechanism: after a user commits to “dark atmospheric,” they may become *better* at distinguishing certain sub-axes that are diagnostic for them (e.g., “cold teal vs warm amber fog”), while becoming *worse* (or indifferent) to axes they treat as irrelevant (e.g., micro-variations in grain pattern). citeturn18view3turn18view4turn4search9

Perceptual learning studies show discrimination thresholds can decrease with practice (i.e., resolution increases), and analyses discuss how learning relates to initial thresholds and scaling functions. citeturn14view3turn4search7

But here’s the cold water: many of these training effects involve **hundreds to thousands of trials**. The Eye Loop should assume within-session learning is limited; instead, it can (a) exploit existing expertise and (b) adapt depth based on observed RT/confidence. citeturn18view4turn14view3

### Expertise: people can discriminate finer within a domain, but it may cost time

Wine expertise studies show experts outperform novices on recognition/identification-type tasks, and some work reports experts can be more accurate yet sometimes slower in response time—suggesting “finer discrimination” can come with more deliberation. citeturn4search1turn4search8

Eye Loop implication: if a user is effectively an “expert” in a visual domain (games UI, film grading, typography), you may see fast confident judgments deeper in the tree; if not, latency will balloon and choices become noisy. Use that as a gating signal: fract deeper only when the user demonstrates discriminability at that level (fast RT, consistent choices). citeturn4search8turn14view3

### How many dimensions and how deep can humans realistically go?

Empirically, even for broad constructs like “aesthetic needs,” factor analyses often yield small numbers of latent factors (e.g., three-factor solutions in one scale; four factors across domains in another). citeturn19view1turn19view2

For applied interface aesthetics, experiments commonly isolate a handful of objective factors (symmetry, complexity, hue, saturation, brightness) that measurably influence subjective aesthetic perception facets. citeturn21view1

On the ML side—directly relevant to eye-loopable visual artifacts—recent work constructing multi-dimensional preference datasets for text-to-image evaluation uses a small set of annotated dimensions (e.g., aesthetics, detail quantity, semantic alignment, overall), explicitly acknowledging that “overall score only” misses important preference structure. citeturn21view4

Put differently: expecting users to navigate 12 truly independent axes inside a single 30-swipe burst is… ambitious. Not impossible for experts, but ambitious.

### IEC fatigue gives you a practical ceiling

Interactive evolutionary computation work repeatedly identifies fatigue as the dominant bottleneck, and reports heuristics like limiting population sizes and generations. In the Picbreeder lineage, the literature cites estimates that a normal IEC process lasts **about 10–20 generations for a single user**, constrained by fatigue. citeturn12view0turn5search2

IEC surveys summarize fatigue quantitatively in terms of number of individuals evaluated, time spent, and number of generations. citeturn18view0

For Eye Loop, the key translation is: **you likely have ~30–40 binary decisions before cognitive fatigue or indifference rises**, unless you’re dramatically shifting cognitive mode (recognition over evaluation, as you established).

## Implementation-ready recipes

### Selecting the next probe with BALD + a fract tree Anima

You asked for a formula that chooses “surface vs depth-fract.” Here is a practical version that cleanly composes:

**State per node \(v\):**
- \(p(v\mid D)\): posterior probability the user intent lives in node \(v\) (branch mass)  
- \(p(\Theta_v\mid D,c_v)\): posterior over node-local parameters  
- \(\mathcal{X}_v\): candidate stimuli generator under node constraints (hold parent invariants)

**Per-stimulus value:**
\[
\text{IG}_v(x) = I(\Theta_v;\, y \mid x, D, c_v)
\]
(use choice-only or choice+RT likelihood; same MI form) citeturn14view1turn0search9

**Global selection rule:**
\[
(v^\*, x^\*) = \arg\max_{v\in\mathcal{F}}
\left[
\frac{p(v\mid D)\;\max_{x\in\mathcal{X}_v}\text{IG}_v(x)}
{\text{Cost}(x)}
\;+\;\text{ExploreBonus}(v)
\right]
\]

Where:
- Cost can be constant per swipe or an RT-predicted cost proxy.
- ExploreBonus can be UCB-style \(c\sqrt{\log N / n_v}\) or Thompson-sampling over “IG-reward,” depending on how bandit-y you want the controller to be. citeturn6search18turn11view2turn0search12

**AMPLe-compatible within-node shortcut (often faster than full MI):**  
Inside a node, you can approximate “best \(x\)” by picking the query whose predicted outcomes are closest to 0.5 (belief-halving), mirroring AMPLe’s acquisition that maximizes the worst-case marginal likelihood (driving both outcomes toward 0.5). citeturn13view1turn13view3

This gives you a two-level controller:

- **outer loop:** choose node \(v\) (depth vs breadth) using weighted IG + exploration bonus  
- **inner loop:** choose stimulus \(x\) using conditional BALD or AMPLe-like halving within that node

That is “BALD meets hierarchical exploration” in one implementable stack.

### Prompting Gemini to vary one orthogonal axis while holding resolved dimensions fixed

The most reliable approach described in Google’s own guidance is: **edit an existing reference image**, instructing the model to change only one element and preserve everything else. citeturn17view1turn17view3turn11view5

Below is a template you can use as a “single-axis variation operator.” (No citations in the template; citations are in the surrounding explanation.)

```text
You are generating VARIANTS of the provided reference image.

INVARIANTS (must remain identical):
- overall mood: [locked parent concept, e.g., "dark atmospheric"]
- composition: [camera angle, framing, layout geometry]
- color palette: [describe]
- lighting direction and contrast: [describe]
- subject identity and key shapes: [describe]
- texture types that must NOT change: [describe]
- typography and spacing rules (if UI): [describe]

ONE VARIABLE AXIS (the only thing you may change):
Axis name: [e.g., "temperature"]
Definition: [what it means in this context]
Allowed range: [e.g., "cold → neutral → warm"]
Discrete levels to generate: L1, L2, L3, L4, L5
For each level, change ONLY what is necessary to express that level.

OUTPUT:
Generate 5 images (L1–L5).
After each image, write a one-sentence "diff" that states ONLY what changed along the axis.
```

Why this works with Gemini specifically:

- Official prompting guidance highlights using image+text editing and even “change only X, keep everything else the same,” including inpainting-style constraints. citeturn17view1turn17view3  
- The Nano Banana guide explicitly recommends reference images + relationship instructions to preserve consistency, which is exactly what “hold resolved dimensions constant” requires. citeturn11view5turn8search17  

Where this breaks down (predictably):

- If your “axis” is correlated in the training distribution with other properties (e.g., “warm” implies “cozy” implies “higher brightness”), the model tends to entangle them. That’s the same failure mode called out in multi-aspect controllable generation research: attribute correlations can cause unintended coupling. citeturn16view1turn16view0  

So the Eye Loop should **test orthogonality empirically**: run the axis operator, then see if user responses treat the axis as independent (consistent preference shifts without collateral drift). If not, the axis is “fake-orthogonal” and should be merged/refined.

### Practical depth limit for a 30-swipe session

A blunt but useful budgeting model:

- A 30-swipe session supports **~2 meaningful fract levels** for most users, occasionally **3** for experts or when the generator’s representation is exceptionally well-behaved.

You can justify that three ways from the literature you’re building on:

1) **IEC fatigue constraints:** single-user interactive evolution is often constrained to ~10–20 generations, with fatigue the limiting factor. Eye Loop reduces per-step burden, but the “human bandwidth ceiling” still exists. citeturn12view0turn18view0  
2) **Control complexity:** multi-attribute control tends to entangle, meaning deeper levels become noisy unless axes are truly separable. citeturn16view1turn16view0  
3) **Discriminability:** category learning can sharpen relevant distinctions but blur irrelevant ones; without extensive training, deeper sub-axis JNDs often won’t be “free.” citeturn18view3turn18view4turn14view3  

A concrete allocation that usually won’t betray you:

- **~10 swipes breadth:** find the top-level region(s) with high posterior mass using high-IG queries.  
- **~15 swipes depth (fract):** pick the best region and resolve ~2 sub-axes (roughly 6–8 swipes per sub-axis including calibration and boundary checks).  
- **~5 swipes refine/lock:** exploit the learned sub-axes to generate “finals,” plus one or two novelty/robustness probes.

Dynamic adjustment using your established RT channel:

- If RT stays fast and consistent at depth, expand one more level (expert-like discriminability). citeturn4search8turn14view3  
- If RT slows and choices become inconsistent, stop fracting and return to breadth or to “produce artifacts” mode (turn decisions into recognition, not analysis). citeturn12view0turn18view0  

If you try to force 4–5 hierarchical levels into 30 swipes, you’re essentially asking the user to become a sommelier of your latent space in under a minute. Sometimes you’ll get a prodigy. Mostly you’ll get fatigue-shaped noise—then your beautiful inference pipeline learns the user’s patience level instead of their taste. citeturn18view0turn12view0
