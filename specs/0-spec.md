# The Eye Loop — Hackathon Build Spec

## Zero to Agent: Vercel x DeepMind | March 21, 2026

**Problem Statement:** Statement Three — AI Applications

**One-line pitch:** A taste amplifier that discovers what you want to build through instinctive selection, not specification.

---

## What It Does

The user types an intent — "weather app for runners." Instead of generating a spec or writing code, the system generates visual facades and the user swipes left or right. Each swipe — plus reaction time — updates a living taste model called the Anima. Agents read the Anima, identify the most uncertain dimension, and generate the next facade to probe it. The process moves from abstract vibes (words, images) to concrete UI (HTML mockups, interactive components). A builder agent assembles surviving artifacts into a coherent prototype continuously in the background. By the time the user finishes swiping, the prototype is already built — composed entirely from what survived selection.

The user never writes a spec. They discover what they meant by watching what they chose.

---

## Core Concepts

**The Eye** — the user. Selects, never specifies. Binary input (swipe) + continuous latency signal.

**The Anima** — the accumulated decision boundary. Not a preference list. A tree of resolved and unresolved dimensions. Every agent reads it before generating. Persona (stated intent) and anima (revealed preference) tracked as separate layers — their divergence is the highest-signal data.

**Facades** — visual artifacts generated to trigger instinctive response. Progress through stages: words → images → HTML mockups → interactive snippets. Each carries a stated hypothesis. Each is generated by a specific agent who receives the swipe result.

**Fracting** — when a dimension resolves, the system identifies orthogonal sub-axes within it and branches. Not deeper on the same axis — orthogonal dimensions hidden inside the resolved one. "Dark atmospheric" fracts into texture × density × temperature. Each axis independent of the others.

**BALD selection** — each facade is chosen not to please, but to maximally partition the remaining uncertainty. Generate the facade where the model's internal hypotheses disagree the most about what the Eye will do.

---

## Agent Roster

**Orchestrator** — Gemini 3.1 Pro. The brain. Manages agent lifecycle, routes swipe results, assigns probe briefs, monitors queue health, triggers compaction. Never generates facades.

**Builder** — Gemini 3.1 Pro. The architect. Runs continuously from first surviving artifact. Maintains a living draft prototype. Identifies construction ambiguities and writes probe briefs. Has the best view of what the system doesn't yet know — because it's the one trying to build with incomplete information. Never generates facades. Never swipe-facing.

**Scout (×3, expandable)** — Gemini Flash for HTML, Nano Banana for images, Veo for interaction clips. The hands. Each runs a Ralph-style loop: generate → push → wait → receive feedback → decide → loop. Pulls probe briefs from the builder first, self-assigns from Anima uncertainty if no briefs pending. Can request child agent spawning when fracting.

**Compactor** — function within orchestrator, uses Gemini 3.1 Pro. Runs every 5 swipes or on contradiction detection. Merges convergent evidence, prunes dead branches, promotes contradictions to new probe briefs, keeps Anima within token budget.

### Communication Model

Agents never call each other directly. Three communication channels:

**Shared context** — the Anima, facade queue, probe queue, draft prototype. All agents read from the same reactive state object. Writes go through context methods that maintain consistency.

**Event bus** — fire-and-forget notifications. `facade-ready`, `swipe-result`, `probe-requested`, `spawn-requested`, `anima-updated`. Agents subscribe to events they care about. The bus also streams to the client via SSE for live visualization.

**Probe queue** — the builder's voice. Builder identifies construction ambiguities, writes detailed probe briefs, pushes to queue. Scouts pull briefs from this queue first (builder-driven, high priority). If empty, scouts self-assign from Anima uncertainty (exploration-driven). The orchestrator never tells a scout what to do — scouts pull work.

Tools go outward (agent → Gemini). Events go sideways (agent ↔ agent, mediated by bus). Shared state goes through the context (everyone reads, specific agents write).

## The Context Class

Single server-side reactive state object. SvelteKit runtime pattern — context class with getters. All coordination logic encoded in derived state.

```
EyeLoopContext
  State:
    .anima            Anima tree (shared read)
    .facades          facade queue (scouts push, client pulls)
    .probes           probe brief queue (builder pushes, scouts pull)
    .agents           registry of active agent states
    .draft            builder's living prototype

  Derived (getters):
    .nextProbe        shift from probe queue, or null
    .mostUncertain    BALD over Anima tree → highest entropy node
    .queueHealthy     facades.length >= 3
    .shouldSpawn      probes.length > active scouts, or queue thin
    .shouldRetire     info gain dropping + queue full

  Methods:
    .addEvidence()    update Anima tree with swipe record
    .pushFacade()     add to queue, emit facade-ready
    .requestProbe()   builder adds brief, emit probe-requested
    .integrate()      builder absorbs surviving artifact into draft
    .compact()        merge evidence, prune, promote contradictions
    .emit() / .on()   event bus
```

The intelligence lives in the getters. `.mostUncertain` implements BALD. `.shouldSpawn` encodes scaling logic. Agents are simple loops. The context is the brain.

## Agent Loops

### Scout Loop (Ralph-style)

```
loop:
  brief ← context.nextProbe OR self-assign from context.mostUncertain
  
  facade ← generate(brief, context.anima)     // tool calls to Gemini
  optionally: critique and refine internally
  
  context.pushFacade(facade)
  result ← wait for swipe on this facade      // event bus subscription
  context.addEvidence(result)
  
  if strongAccept AND orthogonalAxesExist:
    emit 'spawn-requested' with identified sub-axes
```

The scout receives its own facade's result. It builds a local history — not just a stateless worker. Over time, scouts develop specialization based on which axes they've probed.

### Builder Loop

```
on every swipe-result:
  if accept: integrate surviving artifact into draft
  if reject: add anti-pattern constraint to draft
  
  ambiguity ← identify what's blocking construction
  if ambiguity exists:
    context.requestProbe(ambiguity)     // high-priority brief for scouts
  
  for each incomplete section in draft:
    if enough Anima context: extrapolate and fill
```

The builder never generates facades. It reads the results and builds. Its probe briefs are construction-grounded: not "color axis unresolved" but "I'm building the header and need to know: fixed position or scroll-away, given the resolved dark atmospheric layout with organic texture."

### Orchestrator Watch

```
on any event:
  if shouldSpawn: spawn scout → enters scout loop
  if shouldRetire: retire least-active scout
  if swipeCount % 5 == 0: compact Anima
  
  on 'spawn-requested':
    for each sub-axis (up to agent limit):
      spawn child scout with axis assignment
  
  freshness check:
    for each queued facade:
      if hypothesis now redundant: drop, notify source scout
```

## Queue Equilibrium

Swipe cadence: ~2-3 seconds. Generation cadence: 1-8 seconds depending on stage. Queue must always have facades waiting.

**Buffer target:** 3-5 facades. Below 3 → spawn scout or increase priority. Above 5 → hold spawning, reduce critique loops.

**Freshness:** On Anima update, score queued facade hypotheses. Drop any whose axis just got resolved. Don't serve stale probes.

**Stage blending:** No abrupt transitions. Late word-stage facades can carry small images. Early mockup-stage can be partial layouts. Transitions driven by Anima resolution density.

---

## Builder-Driven Probes

The builder has the best view of where ambiguity matters. Abstract uncertainty in the Anima tree is less valuable than construction-grounded uncertainty in the builder's draft.

Builder briefs carry resolved constraints as context, so the scout generates facades the user can judge in context, not in the abstract. When the builder is stuck, all scout energy goes to unblocking it. When the builder has no ambiguities, scouts explore freely.

---

## Agent Spawning

**Fract trigger:** Strong accept → scout emits spawn-requested with orthogonal sub-axes. Orchestrator spawns child scouts.

**Demand trigger:** Probe queue deeper than active scout count. Builder asking questions faster than scouts answer.

**Uncertainty trigger:** Orchestrator finds uncovered high-entropy region in Anima.

**Retirement:** Queue buffer full + information gain dropping → retire, don't spawn.

```
Orchestrator
  ├── Scout A (mood) → resolved "dark atmospheric"
  │     ├── Scout A1 (texture) → resolved "organic"  
  │     │     ├── Scout A1a (rough vs smooth) — active
  │     │     └── Scout A1b (static vs animated) — active
  │     ├── Scout A2 (density) — active
  │     └── Scout A3 (temperature) → resolved "cool"
  ├── Scout B (layout) — working builder brief
  ├── Scout C (interaction) — waiting for mockup stage
  └── Builder — 3 sections complete, 2 ambiguities pending
```

---

## Anima Compaction

Every 5 swipes or on contradiction:

- **Merge** convergent evidence into single resolved nodes
- **Prune** dead branches from strongly resolved axes
- **Promote** contradictions to new probe briefs (axis was misidentified — re-probe with different framing)
- **Summarize** to stay within token budget

---

## The Living Prototype

The builder doesn't assemble at the end. It grows the prototype behind every swipe.

**Three inputs:** Surviving artifacts (material), the Anima tree (rules), rejected artifacts (guardrails).

**Gap filling:** Unresolved sections filled by extrapolation from Anima. The taste profile is a gravitational field. The builder fills the dark matter.

**Reveal:** User finishes swiping. Prototype is already there. Grown, not generated.

## Data Structures

```
Anima
  .intent         original user input (persona)
  .tree           AnimaNode[] — hierarchical preference tree
  .swipeCount     
  .stage          words | images | mockups | components

AnimaNode
  .axis           "aesthetic.mood.texture"
  .resolved       boolean
  .value          "organic" (if resolved)
  .confidence     0-1, from swipe latency
  .children       AnimaNode[] — orthogonal sub-axes
  .evidence       SwipeRecord[]

SwipeRecord
  .facadeId       
  .agentId        source agent
  .hypothesis     what was being tested
  .decision       accept | reject
  .latencyMs      boundary proximity signal

Facade
  .agentId        source agent (receives result back)
  .stage          word | image | mockup | component
  .hypothesis     non-negotiable
  .content        text | base64 image | HTML string
  .axisTarget     which AnimaNode this probes
  .brief          originating ProbeBrief if builder-driven

ProbeBrief
  .source         builder | scout
  .priority       high | normal
  .brief          construction-grounded question
  .context        relevant resolved constraints
```

## Facade Stages

**Words (1-6):** Single words, near-instant. Broadest partitioning.

**Images (7-15):** Nano Banana ~1.5s. Moodboards, palettes, visual concepts. Fracting begins.

**HTML Mockups (16-28):** Gemini Flash, sandboxed iframes. Full styled pages. Builder issuing probe briefs.

**Interactive Snippets (28-35):** Surviving sections promoted to interactive. Veo motion clips. Swipe informed by contact.

**Reveal:** Information gain drops. Builder presents assembled prototype.
